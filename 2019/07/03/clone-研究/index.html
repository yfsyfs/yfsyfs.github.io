下面再讲一下clone复制的深浅性的问题. 首先开宗明义:

所谓浅拷贝就是一个对象占据的堆内存的内部的引用类型（例如String、Date）指向的别的堆内存不会重新拷贝一份，但是基本类型是会重新拷贝一份的. 所以克隆对象对基本数据类型的属性的更改不会影响到原对象. 但是对引用类型属性的更改会影响到原对象的该属性. 但是对于String 这种不可变的引用类型，浅拷贝其实没问题. 因为不论是克隆对象还是 原对象都不能更改它嘛~ 所以其实大家的String属性（其实就是指针）都指向同一块堆内存是没有问题的. 但是如果可变的话就有问题了，一般这样的话，就要再次调用该引用类型属性的clone方法实现对象深复制. 这一点见最后的例子.

clone 拷贝是浅拷贝，如果要实现深拷贝，要自己递归调用clone实现.

注意到clone方法是protected的，这就意味着我们不能直接调用一个未覆写clone方法的对象的clone方法（否则编译都会报错——clone方法is not visible）. 而且根据上面的API注释，这个对象还必须要实现Cloneable接口（一个没有任何实际方法的标记接口）.

如果我们要调用一个类的对象从Object继承来的clone方法，该类必须满足

  1. 对象的类实现了java.lang.Cloneable接口，不然运行时报CloneNotSupportedException
  2. 对象的类覆写了clone方法，不然编译报错——clone is not visible
